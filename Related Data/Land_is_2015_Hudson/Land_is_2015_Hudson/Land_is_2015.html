Description:
Abstract:
Three classes of impervious surfaces--buildings, roads, and other impervious--were mapped for New Jersey through a semi-automated process developed using eCognition software. The automated feature extraction workflow used a Geographic Object-Oriented Image Analysis (GEOBIA) framework to extract the three impervious classes from the source datasets which include digital imagery, LiDAR point clouds and several vector data sets including Land use/land cover, road centerlines and hydrographic features, using a rule-based expert system.
Purpose:
This data set was generated to provide a statewide vector layer of impervious surfaces for New Jersey.
Supplemental_Information:
The project under which this data set was produced was undertaken to investigate the feasibility of using automated object-based image analysis extraction routines to generate a vector layer of impervious surfaces for New Jersey. The 2015 leaf-off New Jersey orthorectified imagery served as the primary data source, but the 1-meter leaf-on imagery acquired through the National Agricultural Imagery Program (NAIP) in the summer of 2015 was also used. Several additional datasets were used to supplement the process. This includes LiDAR data, which was collected at varying time periods in New Jersey, land use/land cover data, road centerlines, and stream and water body features included in the National Hydrography Dataset (NHD). Initial feature extraction was carried out in the coordinate system of the LiDAR data (most often UTM). LiDAR point clouds were normalized and classified to separate out buildings and trees. Raster surface models and intensity images were generated from the LiDAR point cloud data. Imagery data were mosaicked, and vector datasets were examined for completeness, accuracy, and topology. Initial analysis was done using eCognition software, with QA/QC and some final data edits done using ArcGIS 10.5.1. Applied Geographics was the primary contractor on the project, with the Spatial Analysis Lab of the University of Vermont serving as a sub-contractor. NJDEP staff were involved in the development of the mapping criteria as well as in the QA/QC of intermediate and final data sets.

Data_Quality_Information:
Attribute_Accuracy:
Attribute_Accuracy_Report:
An Accuracy Assessment was conducted following manual correction of the impervious surface layer. UVM SAL developed an error matrix with producer, user, and overall error metrics by comparing a setof known sample points to corresponding locations in the respective impervious surface. A systematic grid was generated based on the areal extent of the unique classes, thus ensuring that the sample size unit (SSU) will represent all classes. SSU grid size is defined by a threshold derived from the frequency histogram for the area of individual class features. A stratified random point-sampling method was used to assign a sample point to each SSU and to create an even spatial distribution of points over the project area. The points was then used to extract the class value from the impervious surface layer. In instances where rare classes are covered by sample points that exceed 50% of the SSUs, the selected points were omitted and replaced by generating random points at an interval that will cover approximately 50% of the SSUs for the class. The overall accuracy of the mapping was 94%. The user's accuracies for Buildings, Roads, and Other Paved were 94%, 87%, and 84% respectively.

Process_Step:
Process_Description:
The approach to mapping involved both automated feature extraction techniques and manual editing of impervious surfaces. The impervious surface classes which were mapped are as follows: Structures (buildings), Roads, Other impervious surfaces (e.g. Parking Lots, Sidewalks, patios, driveways, etc.). The University of Vermont (UVM) Spatial Analysis Lab (SAL) designed, developed, and deployed a Geographic Object-Oriented Image Analysis (OBIA) framework that automatically extracts the impervious classes from the source datasets using a rule-based expert system. The expert system made use of segmentation, classification, and morphology algorithms to group pixels into objects, classify the objects based on the properties of the source datasets, and then refine the objects to enhance aesthetics and improve visual realism. A 'bottom up' approach to mapping was carried out in which impervious surface features that are under tree canopy but visible in the leaf-off imagery was mapped. The key advantage of the object-based approach is that the use of the spectral, textural, geometric, and contextual properties in the imagery reduce confusion between impervious land cover features and those that are similar in tone, such sand on the banks of streams. The project team worked with DEP staff to identify extents for a pilot project. The pilot project was utilized to refine general GEOBIA processes being run with New Jersey specific data, to ensure that the final statewide dataset would meet the DEP's expectations. The pilot project also helped identify any specific issues related to the data processing in advance, for discussion and resolution with DEP prior to full statewide processing. During the pilot phase, the GEOBIA processes and analysis were developed with the specific New Jersey data, and carried out against the pilot areas. Upon final approval of the pilot datasets, the project team began statewide processing, utilizing the findings from the pilot project to apply to the statewide processing efforts. With the large area of the State of New Jersey, the project team broke the initial processing down by LiDAR project collection areas, of which there were 11. These areas were processed based on the procedures determined during the pilot project and then delivered to NJ DEP for review. Any issues found during the review period were then revised, and resubmitted to DEP. Upon approval of all areas by DEP, the areas were then combined into one statewide dataset.
To improve the display of the dataset in GIS applications used within NJDEP, a municipal boundary vector data set was integrated into the IS feature class. This was done primarily to break up large continuous road polygons into more manageable features.
